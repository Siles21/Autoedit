Premiere · Sub-Animations Kit

Vom fertigen Cut
zu animierten Overlays.

Aus einem geschnittenen Premiere-Video + Transkript baut Autoedit automatisch frame-genaue Grafiken — Zahlen-Cards, Aufzählungen, Reveals, Lower-Thirds — und legt sie zurück in den Schnitt. Lokal, eigenständig, mit eigener Marke.

Inputfinal.mp4 + Transkript
→
Whisper · Plan · Remotion
→
OutputOverlays im Schnitt
01

Installation

Ein Befehl installiert alles — Python-Umgebung, Render-Engine und die Live-Premiere-Bridge. Getestet auf macOS.

Python ≥ 3.10
brew install python
Node ≥ 18 + npm
brew install node
ffmpeg
brew install ffmpeg
Premiere Pro
nur für Live-Weg
1

Repo klonen

Hol dir das Kit und wechsle hinein.

git clone https://github.com/Siles21/Autoedit
cd Autoedit
2

Setup ausführen

Legt eine isolierte .venv an, installiert alle Python- + Node-Pakete und die Live-Bridge — und prüft sich selbst. Ohne Live-Teil: ./setup.sh --no-live.

./setup.sh
3

Umgebung aktivieren

Einmal pro Terminal — danach laufen alle python3 scripts/…-Befehle.

source .venv/bin/activate
02

So funktioniert's

Sechs Stufen vom Video zur fertigen Grafik-Spur. Stufe 3 ist der einzige Pflicht-Stopp: du gibst den Plan frei.

STUFE 1
Align
Whisper-Timecodes gegen den finalen Cut
STUFE 2
Plan
Zahlen, Listen, Reveals werden erkannt
STUFE 3
Freigabe ✓
Du prüfst & korrigierst den Plan
STUFE 4
Render
Transparente ProRes-Overlays (Remotion)
STUFE 5
Place
Live in Premiere ODER als XML-Export
STUFE 6
QC
Frames prüfen — sitzt alles?
03

Verwendung

Stufe 1–4 sind für beide Wege identisch. Erst bei der Platzierung (Stufe 5) wählst du: bequemer Export oder volle Live-Steuerung.

# im Repo, venv aktiv:
cd skill

# 1 · Whisper-Timecodes
python3 scripts/align_words.py final.mp4 --out work/v1/words.json

# 2 · Plan-Entwurf (danach von dir/Claude vervollständigt)
python3 scripts/build_plan.py work/v1/words.json --out work/v1/animation-plan.json

# 4 · Overlays rendern (eigene Marke via --brand)
python3 scripts/render_overlays.py work/v1/animation-plan.json \
    --remotion remotion --formats 16x9 --out work/v1/overlays --brand default
Stufe 5 · Weg A

Export

Erzeugt FCP7-XML + Platzierungs-Liste. Du ziehst die Overlays in Premiere. Kein Plugin nötig.

python3 scripts/export_fcp7xml.py \
  work/v1/animation-plan.json \
  --video final.mp4 \
  --overlays work/v1/overlays/overlays-manifest.json \
  --out work/v1 --format 16x9
→ einfachster Weg, läuft überall
Stufe 5 · Weg B

Live

Setzt die Overlays direkt ins offene Premiere-Projekt — über die mitgelieferte adb-mcp-Bridge.

# einmalig, dann Premiere neu starten:
./vendor/adb-mcp/uxp-cli.sh enable

# pro Session:
./vendor/adb-mcp/bridge-up.sh

# platzieren:
node vendor/adb-mcp/premiere_drive.js place \
  work/v1/overlays/overlays-manifest.json --vtrack 9
→ Details: docs/LIVE-PREMIERE-SETUP.md
⌘

Eigene Marke: Farben & Schrift kommen aus skill/remotion/brands/<name>.json. Kopiere default.json, passe die Farben an und render mit --brand meinebrand. So tragen alle Overlays dein CI.
